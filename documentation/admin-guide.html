Administrator Overview

As a system administrator, you have full control over Client Approval & Sign-Off Management System. This guide covers:

  • User Management: Create, edit, and manage user accounts
  • Roles & Permissions: Configure access control and permissions
  • Email Management: Customize templates and configure SMTP
  • Audit Logs: Monitor system activity and user actions
  • System Security: Manage security settings and policies

Important Notice

Only users with the Administrator role have access to these features. Use administrative privileges responsibly.

User Management

Creating a New User

  1. 1 Navigate to "Users" in the admin sidebar
  2. 2 Click "Add User" button
  3. 3 Fill in user information
  4. 4 Assign a role (Administrator, Manager, or User)
  5. 5 Click "Create User"

Auto-generated Password: The system can auto-generate a secure password. The user will receive a welcome email with login credentials.

Managing Existing Users

Edit User

Update name, email, or role

Deactivate

Suspend user access temporarily

Reset Password

Send password reset email

Warning: Deleting Users

Deleting a user will permanently remove their account and all associated data. Consider deactivating instead to preserve records.

Email Template Management

Customize email notifications sent by the system. Client Approval & Sign-Off Management System includes 5 default templates:

Welcome Email

User Registration

Sent to new users when their account is created

Available Variables: {name}, {email}, {password}, {app_url}

Password Reset

Password Recovery

Sent when a user requests a password reset

Available Variables: {name}, {reset_url}, {expires_in}

Approval Granted

Approval Success

Sent to client when their request is approved

Available Variables: {client_name}, {approval_title}, {approval_message}, {approved_by}, {app_url}

Approval Rejected

Approval Denied

Sent to client when their request is rejected

Available Variables: {client_name}, {approval_title}, {rejection_reason}, {rejected_by}, {app_url}

New Approval Request

Notification

Sent to administrators when a new approval is created

Available Variables: {client_name}, {approval_title}, {priority}, {due_date}, {view_url}

Editing Email Templates

  1. 1. Click "Email Templates" in the admin menu
  2. 2. Select the template to edit
  3. 3. Modify the subject and body (HTML supported)
  4. 4. Use variables in {curly_braces} format
  5. 5. Click "Save Template"
  6. 6. Send a test email to verify changes

Email Configuration (SMTP)

Configure SMTP settings to enable email notifications. Edit your .env file:

MAIL_MAILER=smtp
MAIL_HOST=smtp.mailtrap.io
MAIL_PORT=2525
MAIL_USERNAME=your_username
MAIL_PASSWORD=your_password
MAIL_ENCRYPTION=tls
MAIL_FROM_ADDRESS=noreply@yourdomain.com
MAIL_FROM_NAME="${APP_NAME}"

Popular SMTP Providers

Gmail

Host: smtp.gmail.com

Port: 587 (TLS) or 465 (SSL)

Note: Use App Password, not your Gmail password

SendGrid

Host: smtp.sendgrid.net

Port: 587

Username: apikey

Mailgun

Host: smtp.mailgun.org

Port: 587

After Configuration

Run php artisan config:cache to apply changes. Test email functionality from Email Templates page.

Audit Logs

Monitor all system activity with comprehensive audit logging. Track user actions, changes, and important events.

What's Logged?

User Events

  • Login/logout activity
  • User creation and deletion
  • Role changes
  • Password resets

Client Events

  • Client created/updated/deleted
  • Status changes
  • Bulk operations
  • Data exports

Approval Events

  • Request created/edited/deleted
  • Approval granted/rejected
  • Status changes
  • Attachments added/removed

System Events

  • Configuration changes
  • Email template updates
  • Role/permission modifications
  • Failed login attempts

Security Settings

Security Best Practices

  • Strong Passwords: Enforce minimum 8 characters with mixed case, numbers, and symbols
  • Regular Audits: Review audit logs weekly for suspicious activity
  • Role-Based Access: Assign minimum necessary permissions to each role
  • HTTPS Only: Ensure APP_URL uses https:// in production
  • Regular Updates: Keep Laravel and dependencies up-to-date

Password Policies

  • Minimum 8 characters required
  • Password reset expires in 60 minutes
  • Failed login attempts logged
  • Passwords hashed with bcrypt

Data Protection

  • All data encrypted at rest
  • HTTPS enforced for transmission
  • CSRF protection enabled
  • XSS protection active

System Maintenance

Cache Management

Clear application cache to apply configuration changes:

php artisan cache:clear
php artisan config:cache
php artisan route:cache
php artisan view:clear

Run these commands after updating .env or configuration files.

Database Backup

Regular backups are critical. Use mysqldump or your hosting panel:

mysqldump -u username -p database_name > backup_$(date +%Y%m%d).sql

Recommendation: Schedule daily automated backups and store off-site.

Updates & Upgrades

Keep your system secure and up-to-date:

  • Test updates on staging environment first
  • Backup database before major updates
  • Run composer update for dependencies
