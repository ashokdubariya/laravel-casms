Installation Guide

System Requirements

Before installing Client Approval & Sign-Off Management System, ensure your server meets these requirements:

Server Requirements

  • PHP: 8.2+ or higher
  • Web Server: Apache 2.4+ or Nginx 1.20+
  • Database: MySQL 5.7+ or MariaDB 10.3+
  • Composer: 2.x
  • Node.js: 18+ with NPM

PHP Extensions

  • BCMath
  • Ctype, Fileinfo, JSON
  • Mbstring, PDO, Tokenizer
  • XML, DOM, OpenSSL
  • GD or Imagick (for images)

Checking Requirements

The Client Approval & Sign-Off Management System web installer will automatically check all requirements when you access it for the first time.

Download & Extract

Step 1: Download Client Approval & Sign-Off Management System

Download the Client Approval & Sign-Off Management System package from your purchase location (email, etc.).

casms-v1.0.0.zip

Step 2: Extract Files

Extract the ZIP file to your local computer:

# On Linux/Mac
unzip casms-v1.0.0.zip

# On Windows
Right-click > Extract All...

Upload Files to Server

Upload all extracted files to your web server root directory:

  • /public_html/ (cPanel/shared hosting)
  • /var/www/html/ (VPS/dedicated server)
  • /home/username/public_html/ (custom setup)

Important Note

The public folder should be your web root. Configure your domain to point to the public directory for better security.

Upload Methods:

FTP/SFTP

Using FileZilla, WinSCP, or similar:

  1. Connect to your server
  2. Navigate to web root
  3. Upload all files
  4. Wait for transfer to complete

SSH/Command Line

Via terminal:

scp casms-v1.0.0.zip user@server:/path/
ssh user@server
cd /path/
unzip casms-v1.0.0.zip

Install Dependencies

Optional: You can skip this section and use the web installer directly. Dependencies can be installed after installation if needed.

If you want to pre-install dependencies, SSH into your server and run:

Install PHP Dependencies 

# Navigate to project directory
cd /var/www/html/casms

# Install Composer dependencies (production)
composer install --no-dev --optimize-autoloader

# For development (includes testing tools)
composer install

Install Node Dependencies & Build Assets 

# Install NPM packages
npm install

# Build production assets
npm run build

# For development with hot reload
npm run dev

Success Indicators

You should see vendor/ and node_modules/ directories created, and compiled assets in public/build/.

Set File Permissions

Laravel requires write permissions for certain directories:

# Make storage and cache writable
chmod -R 775 storage bootstrap/cache

# Set proper ownership (replace www-data with your web server user)
chown -R www-data:www-data storage bootstrap/cache

# Alternative: using current user and web server group
chown -R $USER:www-data storage bootstrap/cache

Security Note

Never set permissions to 777 in production. Use 755 for directories and 644 for files, except for storage and cache which need 775.

Recommended permissions:

  • Directories: 755
  • Files: 644
  • storage/ and bootstrap/cache/: 775

Web Installer Wizard

Client Approval & Sign-Off Management System includes a user-friendly web installer that handles all configuration automatically.

Execute the below commands to start the installation steps.

# Copy the example environment file to create your actual .env file
cp .env.example .env

# Generate the application encryption key
php artisan key:generate

# Clear and refresh the configuration cache.
php artisan config:clear

Simply navigate to your domain to begin:

https://yourdomain.com/installer
1

The installer will automatically verify:

  • PHP version (8.2+)
  • Required PHP extensions
  • Directory permissions
  • Apache/Nginx configuration

Database Configuration

Enter your database details:

localhost or 127.0.0.1

Post-Installation Configuration

Email Configuration (SMTP)

Edit .env file to configure email sending:

MAIL_MAILER=smtp
MAIL_HOST=smtp.gmail.com
MAIL_PORT=587
MAIL_USERNAME=your-email@gmail.com
MAIL_PASSWORD=your-app-password
MAIL_ENCRYPTION=tls
MAIL_FROM_ADDRESS=noreply@yourdomain.com
MAIL_FROM_NAME="${APP_NAME}"

Application URL

Set your application URL in .env:

APP_URL=https://yourdomain.com

Cache Configuration

After any configuration changes, clear the cache:

php artisan config:clear
php artisan cache:clear
php artisan route:clear
php artisan view:clear

# Or clear all at once
php artisan optimize:clear

Common Installation Issues

500 Internal Server Error

Cause: Permissions or .htaccess issue

Solution:

# Fix storage permissions
chmod -R 775 storage bootstrap/cache
chown -R www-data:www-data storage bootstrap/cache

# Ensure .htaccess exists in public/ directory
# Enable Apache mod_rewrite module
# The installer will create .env automatically

Blank White Page

Cause: Missing .env file or APP_KEY

Solution:

# Simply access the web installer:
# Visit: https://yourdomain.com/installer
# The installer will create .env automatically

# If that doesn't work, check permissions:
chmod -R 775 storage bootstrap/cache

Database Connection Failed

Cause: Incorrect credentials or database doesn't exist

Solution:

  • Verify database credentials in .env
  • Ensure database exists (create it if not)
  • Check if MySQL/PostgreSQL is running
  • Test connection: php artisan db:show

Assets Not Loading (CSS/JS)

Cause: Assets not compiled or wrong path

Solution:

npm install
npm run build
# Verify APP_URL in .env matches your domain
php artisan config:clear

Still Having Issues?

Enable debug mode temporarily to see detailed errors:

APP_DEBUG=true  # In .env - ONLY for troubleshooting!

Remember: Always set APP_DEBUG=false in production!
